Latest Source Code

On a daily basis, the software developers upload their latest versions of the source code here. On a weekly basis, the build engineer downloads all these modules, unzips each module in its own subdirectory and then does a build.

  • Common: This C++ module includes only the common header file(s) which are included by all the C++ modules.
    [Author Name]
  • Control: This C++ module is used to build the Control DLL.
    [Author Name]
  • Engine: This C++ module is used to build the Engine DLL.
    [Author Name]
  • Report: This C++ module is used to build the Report DLL.
    [Author Name]
  • Workspace: This module contains the Visual Studio workspace (DSW) file used to build all the DLL's.
    [Author Name]
  • GUI: This Visual Basic module is used to build the Graphical User Interface (GUI) executable.
    [Author Name]
  • Install: This module is used to build the installation package. It includes a Microsoft Visual Studio solution and Visual Studio Installer file. Other files that need to be included in the installation package (e.g. images, SUM, etc.) are also included in this module.
    [Author Name]

* Note Regarding Source Code: A source code entry in the Builds table above provides a "snapshot" of the versions of source code used that week for the build. For example, for the first build, the build engineer downloads all the latest source code zip files (see the "Latest Source Code" section) and then saves them into one big comprehensive zip file and names it "source_1.zip". For the second build, we again save all the source code zip files listed above into one zip file and name it "source_2.zip". And so on.

** Note Regarding Patches: After a given build is released, and before the next build is done (i.e. during the week between builds), developers may fix (patch) their module and rebuild their component (e.g. a patched DLL). The developer will add the component to the appropriate patches ZIP file and upload the ZIP file for other testers to take advantage of the fix before the next build is done.

*** Note Regarding Build 8: Typically, Build 7 is used for the Qualification Test (QT) on session 8 of this course. You may consider Build 7 as the final version. Or, after the conclusion of the QT, your team may want to generate one last build to use for the final demo of the project on session 9 of this course. Build 8 is optional. Build 8 is the absolute final build of the product. If you do not generate Build 8, then remove this row from the table above.
