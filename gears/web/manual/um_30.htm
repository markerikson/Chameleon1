3.0 Network Connectivity

Chameleon provides you with a Network Transparent environment that allows for development and testing of code on the computer infront of you and on a remote server without the user having to know about the difference.

3.1 Remote vs Local

A remote file is a file that is stored on a remote server. A local file is a file that is stored on the computer in front of you.

All actions have equivalent local and remote abilities. The controls are the same, so your interaction is irrelevant of locality.

Most menu and toolbar actions are preformed on the file currently being edited. If the current visible file is a local file, compliation, debugging, and execution will be run locally. If the current visible file is a remote file, compliation, debugging, and exection will be run on the remote server.

3.2 How to connect

Opening or saving to remote files can be accomplished through the appropriate menu commands, after a correct username, hostname, and password have been entered in the options menu.

Opening local files can be accomplished whenever, if you have the proper permission to do so.

3.3 Common Networking Error Messages

"Unable to authenticate." or "Authentication Failed"
his means that the username and password that you used were not accepted on the server you tried connecting to.
"Permssion Denied."
You cannot save the file to the desired directory because it is write protected, or use filename you chose because there already exists a file with that name that you may not overwrite because of write protection.
"Read Denied"
You do not have permission to read the directory or file you are trying to access.
"Connection Refused"
The server you are trying to connect to did not allow you to connect. Usually a result of the server not having an SSH service running.
