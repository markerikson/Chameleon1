Tutorial

Introduction

Welcome to Chameleon, the C++ editor designed for beginning programming students. This tutorial is designed to familiarize you with Chameleon, show you some of its features, and give you a feel for how it is used. Whenever your professor enables additional features, come back to this section for additional tutorials.

Running Chameleon for the first time

Let's start with the basics. First, run Chameleon. Depending on your installation options, there should be a "Digital Gearheads" group in your Start Menu, and (if you selected it) a Chameleon icon on your desktop.

Upon first glance, Chameleon may look pretty simple. In fact, it probably doesn't look a whole lot different than Notepad or Wordpad. However, Chameleon is far more powerful than either of those. It has a variety of features that your professor will enable as you get farther into the semester.

The main area on the screen is the text editor, where you'll write your code. Unlike Notepad, Chameleon lets you have multiple files open at once, each in its own tab. Right now there is one file open, which is lableled "(?) <untitled> 1". When you save or open a file, Chameleon will put its name on that file's tab. We'll get to what the question mark means in just a minute. First, try starting a file in a new tab. To do this, you can either click the "New" button on the left side of the toolbar, or going to the File menu and selecting "New Source File". A second tab will pop up in the tab list, with the text "(?) <untitled> 2".

With the second tab active, click in the text area and try typing something. An asterisk ("*") should appear at the end of the tab label. This means there have been changes to that file since it was last saved. Click the tab labeled "(?) <untitled> 1". The text in the window disappears, because the first file didn't have anything in it. Click in the text area and press the Enter key a few times. Each line in the editor is marked on the left side. As you program, you'll quickly learn that line numbers can greatly help you with your programming.

Code files are pretty useless until they're saved. Save your file by either clicking the "Save" button on the toolbar, or going to the File menu and selecting "Save Source File". A small window will pop up, asking you whether you want to save this file remotely or locally. That's our cue to pause and learn about one of the things that makes Chameleon unique.

Most programs you use keep all their files on whatever computer you happen to be at. However, as a beginning programming student, most of your code files will be kept on a Linux server somewhere on the campus network. Chameleon lets you open and save files on either the remote server or on the local computer. Chameleon keeps track of where each file was last saved. You can easily see where a file was last saved by looking at the label on that file's tab. If the file was last saved to the remote server, there will be an 'R' before the file's name. If the file was last saved on the local computer, there will be an 'L' before the file's name. However, for a new file that hasn't been saved yet, there will be a '?'. Once the file has been saved, that will change to either an 'L' or an 'R'. Now that you know about local and remote modes, let's pick up where we left off.

You should still have a dialog on your screen asking where you want to save this file. Choose "Remote" by either double-clicking the line in the box, or clicking the item once and clicking "OK". After a second, a message box will pop up, saying that Chameleon couldn't log you in using the current username and password. Click "OK", then we'll take care of that.

Click the "Tools" menu, then click "Options". This will pop up Chameleon's options dialog. Ignore the "Features" tab for now. Click the tab labeled "Network". On this tab are four text fields. At the moment, they should be blank. If this information is not filled in correctly, Chameleon can't do anything with files that are on a remote server. Your professor should have given you the information you need to fill in these fields. Go ahead and do that now, then click "OK".
