3.0 Naming and Coding Standards

3.1 Naming Standards

Our naming standards should follow standards set by the Cedarville Computer Science Dept., tentatively called The C++ Style Guide. The specifics are thus:

  • File Name Styles: file name must end in an appropriate standard extension (eg: .cpp for source files).

  • File Name Length: no specifics

  • Function Names: descriptive, with the first letter lower-case, but every new word's first letter upper-case.

  • Class Names: same as Function names, only with the first letter being upper-case.

  • Global Data names: global anything is highly discouraged, but if used, global data will be clearly labeled as such.

  • Local Data names: same as function names.

3.2 Coding Standards

Our coding standards should follow standards set by the Cedarville Computer Science Dept., tentatively called The C++ Style Guide. The specifics are thus:

  • Function prototypes are encouraged.

  • Pre- and Post-conditions for each function are to be specified with specifics regarding variable use.

  • Liberal use of white space and paranthesis are encouraged.

  • Descriptive, not exhaustive, commenting is encouraged.

  • A consistent indentation to be used for each block.

  • Each file shall have a generic header including:

    • Name
    • Date of creation (and completion if desired)
    • Author's name
    • General description of the purpose of the file
    • Revision listing
