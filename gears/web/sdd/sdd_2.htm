2.0 Development and Execution Environment

2.1 Development Environment

2.1.1 Hardware

Chameleon is being developed on standard x86-based computers. Specifically, we are using i686 processors with clock rates at or above 1 gigahertz with a minimum of 256MB of memory.

2.1.2 Software

Development will be done under Windows using a variety of software tools.  These may include Microsoft Visual Studio versions 6 and 7 and their related C++ compilers; the vi editor and the mingw C++ compiler; the CVS server and various CVS clients, including TortoiseCVS; and various other utilities as necessary.

Some development may also occur under Linux using Eclipse 2.1 and gcc.

The wxWindows Windowing Library needs to be installed on the development computers to compile against it.

2.2 Execution Environment

2.2.1 Hardware

Chameleon will run on standard x86-based computers.  It requires a network connection for remote access.  Hardware problems are not likely to be an issue.

We will test functionality on hardware identical to the Development Hardware: i686 processor, 1 gigahert clock or faster, and 256MB of memory or more.

2.2.2 Software

Chameleon should run under any 32-bit Windows operating system, from Windows 95 to Windows XP. 

The wxWindows windowing library will not need to be installed on the Execution machines because it will be statically linked into Chameleon.
