1.0 Introduction

1.1 Purpose of Document

Using a Configuration Management system is vital because it allows users to keep frequent incremental backups of new code, keep track of changes, fascilitate concurrent development, and organize releases. As a result, we intend to use the Configuration Management system CVS in the Chameleon project. This document outlines our plans for using CVS.

1.2 Scope

The Chameleon IDE is to have variable functionality based on master-control input of a fully functioning IDE. Anything that is within a generic IDE is fair game for the scope of this project, up to and including debugging, compiling, and editing. The scope is increased with code completion, the presence of a terminal window, project management, and various code-editing features.

The scope of this document includes all team members and their responsibilities relating to Configuration Management.
