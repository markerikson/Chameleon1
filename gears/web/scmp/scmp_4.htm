4.0 Activities

4.1 Configuration Identification

The standard used to name CFs is the filename of the item with respect to the root SCML directory. Forward slashes are used to denote subdirectories. For example, the single string "images/readme.txt" names a specific Configuration File in the Software Configuration Management Library. All Configuration Files will be listed in the file chameleon_files.txt.

4.2 SCML Location

During the development of the project, the SCML is stored on one of the Cedarville CS Department servers. The exact location will be determined by the Lab Administrator.

4.3 Configuration Control

We will be using various CVS clients, depending on needs and current working platform. Under Windows, this may include WinCVS and TortoiseCVS; under Linux, this may include LinCVS, Cervisia, and the command-line CVS client.

We will be maintaining one SCML, using CVS. To this, the SCM Users will commit all code changes. At the end of each iteration, after advisement from the Test/QA Manager, the SCM Administrator will cvs-tag the code as a stable build release.

CVS provides the ability to record comments when making a checkin. Each checkin has its time and author listed as well. This will serve as our main means of tracking changes. Further procedures will be worked out at future meetings.

We intend to maintain a policy of individual code ownership. This should minimize conflicts involving multiple developers working on the same code. With that said, CVS is designed to handle multiple users working on the same file simultaneously, and is capable of intelligently merging file differences. Should more than one developer be working on a file at the same time, problems should be minimal.

4.4 Configuration Reviews/Audits

The SCM Administrator will review the SCML on a bi-weekly basis (ie, at the end of each iteration), during the integration of code into the stable branch.

We have node determined if we will have code reviews before the tagging of a build.

4.5 SCML Archives

A current copy of the stable branch of the SCML will be included in each bi-weekly release, in addition to the binary executables.
