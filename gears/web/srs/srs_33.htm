3.3 Performance Requirements

This section describes the performance requirements of Chameleon with regard to Operational Requirements, Exception Handling, and Testing.

3.3.1 Operational Requirements

Chameleon is an IDE. As such, there are few operational requitements. Obviously the GUI will be expected to have reasonable responsiveness. The network interactions will also not be very demanding.

3.3.2 Exception Handling

Chameleon will attempt to handle errors internally as much as possible without notifying the user. Some errors may result in unavoidable warnings to the user, such as C++ exceptions internal to the Chameleon code.

3.3.3 Testing Requirements

Besides the usual bug-fixing that goes on while writing code, further testing with students in CS1210 is suggested to gain user feedback.
