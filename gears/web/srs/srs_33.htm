3.3 Performance Requirements

This section describes the performance requirements of Chameleon with regard to Operational Requirements, Exception Handling, and Testing.

3.3.1 Operational Requirements

Chameleon is an IDE. As such, there are few operational requitements. Obviously the GUI will be expected to have reasonable responsiveness. The network interactions will also not be very demanding.

3.3.2 Exception Handling

The most prevelant exceptions that will be dealt with will be compilation errors, which really are not exceptions. The most user induced exceptions will deal with file input and output, whether trying to read from inaccessible files, or attempting to write to inaccesible locations. These exceptions will be reported to the user as soon as they occur allow the user to correct the mistake, and deal with the exception..

3.3.3 Testing Requirements

Because this product will be cross-platform (Linux and Win32 based), and functional in both a networked environment and a non-networked environment, all four combinations of these will need to be tested.
