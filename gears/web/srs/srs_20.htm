2.0 General Description

2.1 Product Perspective

The goal of the Chameleon project is to create a programmer's editor designed for beginning programming students that can adapt to their needs as they gain experience, as well as gradually introduce them to advanced features found in modern IDEs such as Microsoft Visual Studio.

2.2 Product Functions and Features

As mentioned in part 1 of this document, the product will present all of the functionality of a generic IDE, as well as certain abilities designed specifically for the project. These features include:
  • A Master Control program from which the instructor can lock and unlock advanced IDE features such as advanced debugging tools, class views or windows, terminal access, project access, etc.
  • A command-line terminal inside the program for communication with a remote server.
  • Debugging ability from base error detection to advanced step-by-step debugging ability.
  • Code syntax completion on-the-fly, completing class or common variable names while the user is typing them (akin to Micosoft Word's ability to complete the date while typing it).
  • Syntax highlighting
  • Code block hiding: hide blocks of code down to one descriptive line.
  • Project handling on a minor level
  • Generic Editor Behaviors

2.3 User Characteristics

The expected entry level user (base level of experience) is that of a student with no experience with any IDE. The teacher will have the master control switch to limit the functionality of the IDE so that a user new to an IDE will not feel overwhelmed with the eyefulls of functionality that a full IDE can contain. On the other side, power users should feel right at home with the IDE being fully able to handle most major programming tasks. Chameleon is expected to cater to brand new users, novice users, moderate users, veteran users, and power users.

Entry level users are just starting to program. The editor will most likely appear as a simple textbox without any coding assistance. All compilation is done manually by the user using the terminal window. The IDE allows the user to open and save files, along with standard clipboard functions, but no further visual aids are provided.

A novice user has learned the basics of programming. Features such as syntax highlighting and debugging are likely to become useful at this stage.

Moderate users are comfortable with the standard tasks involved in writing a program. They will most likely be ready to begin somewhat larger projects. Chameleon's project management will allow multiple files and the inclusion of outside libraries. Since the user is now familiar with standard tasks such as compiling, they no longer need to be done by hand and can be done with a standard toolbar button instead.

Advanced users are completely comfortable with most of the aspects of a programming project. At this point, the user has access to all of Chameleon's available features, such as code folding in the editor.

Here's an example scenario in which the product is used:

Jay R. Student is a freshman in CS1210.  He's never done any programming before, but he considers himself a reasonably advanced Windows user.  For his first programming assignment, he starts Chameleon and logs into John using the terminal.  Jay decides to start with the "Hello World" example from class to make sure he knows what he's doing.  He types it in, then goes to save it.  When he hits save, the dialog shows the contents of his home folder, where Jay saves the file as "hello.cpp".  Jay then clicks on the terminal window, and following his instructions types in "g++ hello.cpp -o hello.exe".  Fortunately, he entered the example correctly and it compiles fine.  Again following instructions, he types in "./hello.exe" to run it and watches it print "Hello World" in his terminal.  Now that he has some idea what he's doing, he goes on to write the rest of his program.

Later in the semester, Jay is working on another assignment.  By this time, more features have been enabled.  As he types in his code, it's automatically highlighted and helps him with proper indentation.  There's now a compile button available on the toolbar.  When Jay clicks it, the command line is inserted into the terminal, but Jay has to hit enter in order to execute it.

2.4 General Constraints

Chameleon's main target platform is Windows. The ability to run under Linux is desired, but as a secondary goal only. Exact specifications as to disk, RAM, Ethernet, operating system, and other general requirements are unknown. The best rule-of-thumb, as with all software, is to have the most up-to-date version of operating system possible. This will take care of minimum disk requirements and memory requirements.

2.5 Assumptions and Dependencies

This IDE is being provided as is, and it is assumed therefore that the actual ability to program has been communicated by some other means. The ability to correctly use our software when fully enabled can be inferred from accompanying documentation. This IDE is being presented under the assumption that it is going to be used in an education environment, though private use is being considered.
