1.0 Introduction

1.1 Purpose of Document

This document is to serve as a preliminary guideline in what the project item, known commonly as Chameleon, is to accomplish to be at least a minimal success. It is to lay forth explicit tasks that Chameleon must accomplish, features it must have, and performances it must meet. This document is in it's preliminary stages, and, as a result, is subject to change.

1.2 Scope

The product is to have variable functionality based on master-control input of a fully functioning IDE. Anything that is within a generic IDE is fair game for the scope of this project, up to and including debugging, compiling, and editing. The scope is increased with code completion, the presence some form of a terminal, syntax highlighting, code hiding, and minor project handling.

Roles with respect to the scope of the project are as follows: Mark will serve as Project and Team Lead; Ben will serve as Requirements Lead, and Testing/Quality Assurance Lead; and, David will serve as Design Lead and Configuration Manager.
