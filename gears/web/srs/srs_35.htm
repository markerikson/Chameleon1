3.5 Attributes

This section describes the product's attributes.

3.5.1 Availability

The product should be considered always available.

3.5.2 Security

No self-security is needed. The program does however need to be able to communicate with a linux server running SSH2, a form of cryptography.

3.5.3 Maintainability

The product does not need to be extensible after completion. Newer features / bug fixes may be added with later release. No ability for coupling or cohesion is being designed in.

3.5.4 Transferability/Portability

The product is being designed to work on the Linux and win32 operating systems. In theory, the product should be portable to other operating systems. Because there is no direct hardware interaction, the product should be able to be ported to other hardware.
